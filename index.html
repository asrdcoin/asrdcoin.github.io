About ASRD

ASRD is a BEP-20 utility token designed to lower the cost of on-chain interactions while providing mechanisms for ongoing ecosystem support. Key mechanics include a transfer burn, a treasury-funded buyback program and developer grant funding. The design intentionally prioritizes utility, transparency and security.

Core Use Cases

  • Payments & micro-transactions inside partner applications
  • Fee-sharing and reward distribution to support holders and contributors
  • Staking & priority access to platform features
  • Developer grants and community-driven integrations
  • Liquidity incentives and wallet integrations

On-transfer mechanics (example)

A small transfer fee helps fund the treasury and create controlled deflation via burning. Fee percentages are configurable within governance limits.

Example: feeBasis = 200 (2.00%)
- burn = 1%  (permanent removal)
- treasury = 1% (ecosystem support & buybacks)

These are defaults and may be adjusted by governance within defined safety caps.

Governance

ASRD follows a hybrid governance model: founder-operated multisig for early operations, with a clear migration path to token-weighted on-chain governance. Critical actions will use timelocks and multisig safeguards to reduce unilateral risks.

Suggested Safeguards

  • Multisig treasury (e.g., 3-of-5 Gnosis Safe)
  • Timelock delay for parameter changes (48–72 hours)
  • Off-chain proposal forum and snapshot signaling before on-chain votes

Governance Scope

Parameter adjustments, treasury allocations, grant approvals, and upgrades — executed under multi-layer checks and public disclosure.

Security & Audits

Security is a top priority: audits, a public bug bounty program and multi-signature treasury controls are planned before major exchange outreach. The smart contract code exists but must complete independent review before major operations.

Planned Actions

  • External audit before listings
  • Bug bounty on mainstream platforms
  • Gnosis Safe multisig for all treasury functions
  • Timelock on critical admin functions
